###分行语句需要“，”

```
return fmt.Sprintf(
	"%s:%s@tcp(%s:%s)/%scharset=utf8mb4&parseTime=True&loc=Local",
	c.DBUser,
	c.DBPassword,
	c.DBHost,
	c.DBPort,
	c.DBName,
)
```
最后一个name后面也要有逗号

当你的函数参数（或者结构体初始化、切片定义等）**分多行书写**时，最后一项后面的逗号是必须填写的。
###  核心原因：自动插入分号机制

Go 语言在编译时，编译器会自动在行尾尝试插入分号（`;`）来简化代码编写。

- **如果最后一项后面没有逗号：** 编译器会在这一行的末尾自动添加一个分号。例如，它会把代码看成 `c.DBName; )`。这样就会导致语法错误，因为右括号 `)` 之前出现分号在函数调用中是非法的。
    
- **如果加上了逗号：** 编译器看到逗号，知道这一项还没完，就不会乱加分号，从而保证了语法结构的正确性。`
