# 工作流
	1: 根据客服描述，去机器找问题日志发生时间并获取bag -> 上升给研发审核-> 如果不是软件问题， 并且用户配合就可以解决。 
	2: 登录电子脚本前需要确认用户是否在跑计划（影响未知，所以跑计划的时候不登录）
	3：根据描述查看app，恢复用户。	
	工具： 查看用户订单记录： ERP 
	4:硬件换机工作流： 如果用户机器可以登录，登录电子脚本诊断机器 -> 确认换机后，提工单-> erp提退换机->回复用户邮件。
		4.2 碰撞问题， 换了碰撞条也无法解决。 换机工单，物流，回邮件。
